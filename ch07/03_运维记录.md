
### 驱动不一致问题

```js
    # mongoexport -h 192.168.2.104:37017 -u plumzero -p mypassword --authenticationDatabase=admin -d admin -c traderDay --type=json --out /tmp/traderDay.json
    2022-03-22T10:00:35.460+0800    error connecting to db server: server returned error on SASL authentication step: BSON field 'saslContinue.mechanism' is an unknown field.
```
表面上根据提示是认证失败。

命令连接的是 docker 容器，其 mongodb 版本较高，而 mongoexport 使用的是本地环境上的，版本较低。

mongoexport 工具换成较高版本的就行了。

### 在 Shell 会话中执行命令

示例:
```js
    mongo --host 192.168.2.104:27017 -u urname -p urpwd --authenticationDatabase admin urdb --eval 'printjson(db.urcoll.find().count())'
```

### 关于 mongoexport 等工具的使用

1. 相比于导出为 .csv 文件，更建议导出为 .json 文件。因为后者具有多样化的类型，不至于将字符串 '000123' 转换为 123 。
2. 使用 mongoexport 导出为文件，再将文件通过 mongoimport 导入到另一个数据库，无法保证两个数据库数据顺序一致。
